Stats about Pokemon Types!

About Website

We set out to create a “Pokédex” using a Pokémon image dataset found at the link . Given the set of images that were all labeled with the different characters names, we created a model to predict the categories for a set of test images and measured the accuracy of the predictions. Included in the test images were viewpoint variation, scale variation, image deviations (drawings, sketches, costumes), illumination differences, etc. These different examples of each image class allowed the computer to develop learning algorithms that look at the different setting and learn about the visual appearance of each class. This dataset of labeled images was fed to the computer to get familiar with the data and is known as Convolutional Neural Networks (CNNs). This model allows for few parameters which reduces the amount of data required to train the model.

The model consists of three convolutional blocks which act as filters that are used to train the model which results in highly specific features that the model detects to classify the image. We used the Adam (Adaptive Moment Estimation) optimizer, which is known to have greater accuracy in classification and segmentation and “Categorical Crossentropy” loss function, which is used when there are two or more label classes. We have 149 classes, or Pokémon types. You can view the model summary below:



To train the model, we split the data into approx. 80% training and 20% validation images to optimize the classification and set Epochs at 15. Images used to test data were taken from URL images online to evaluate the accuracy of the model based on images the model had not been exposed to yet. This allowed us to determine the number of epochs needed based on the ratio of validation accuracy and training accuracy.
